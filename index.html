Datahub API Architecture Draft

This is the interface standard designed for your future migration of study materials from local storage to a cloud database hosted on your Ubuntu VPS.

POST /api/v1/files/upload

Upload new study materials (supports PDF, Docx, Images, etc.).

// Request Body (FormData)
file: (Binary Data)
lv_nummer: "51PV19441" // Linked course ID
folder: "skripte" // Directory category
GET /api/v1/files/:lv_nummer

Fetch all uploaded files metadata for a specific course.

// Response Example
{
  "status": "success",
  "data": [
    { "id": "uuid-123", "name": "Mechanik_Skript.pdf", "url": "https://zemingwang.de/uploads/uuid-123.pdf", "size": 2048576 }
  ]
}
Tech Stack Recommendations (for Ubuntu VPS):
• Backend Framework: Node.js (Express) or Python (FastAPI). Both are lightweight and perfect for REST APIs.
• Database: PostgreSQL (highly scalable) or SQLite (zero-configuration, single file).
• File Storage: Actual PDFs are stored on the VPS disk (e.g., /var/www/uploads/), while the database only stores the metadata and file paths.
